# ⚖️ 你的工作区协议 (PROTOCOL)

> **性质**：本文件是工作区宪法。所有项目、所有 Agent、所有协作者必须遵守。
> **违反后果**：变更不得合入、检查点拒绝保存、Agent 必须拒绝继续。

---

## §1 项目文件三件套（强制）

每个项目目录下**必须**包含以下三个文件，缺一不可：

| 文件 | 作用 | 谁来读 |
|------|------|--------|
| `CLAUDE.md` | 项目操作规范（自动加载） | Claude Code Agent 进入目录时自动读取 |
| `PROGRESS.md` | 进度追踪（机器可解析） | Hook 脚本 + Agent 变更后自动写入 |
| `GATES.md` | 质量门禁清单 | 完成阶段前逐项核对 |

> 目录下缺少任一文件 → Agent 必须**先补齐再工作**，不得跳过。

## §2 变更三阶段（不可跳过）

```
  ┌──────────┐     ┌──────────┐     ┌──────────┐
  │ ① BEFORE  │ ──▶ │ ②  CHANGE │ ──▶ │ ③ AFTER   │
  │ 读取规范  │     │ 执行变更  │     │ 更新进度  │
  └──────────┘     └──────────┘     └──────────┘
```

### 阶段 ① — BEFORE（强制前置）
1. **读取项目的 `CLAUDE.md`** — 理解项目身份和约束
2. **读取项目的 `PROGRESS.md`** — 了解当前进度和上下文
3. **读取项目的 `GATES.md`** — 知道本次变更必须通过哪些门禁
4. **更新 `PROGRESS.md`** — 将当前任务标记为 `🔄 进行中`

### 阶段 ② — CHANGE（执行）
执行实际变更。遵循 `CLAUDE.md` 中的项目特定规则。

### 阶段 ③ — AFTER（强制后置）
1. **运行门禁检查** — 逐项对照 `GATES.md`
2. **更新 `PROGRESS.md`**：
   - 标记任务为 `✅ 已完成`
   - 更新 `最后更新时间`
   - 追加变更摘要
3. **触发检查点保存** — 调用 save-checkpoint hook

## §3 PROGRESS.md 格式规范（机器可解析）

```markdown
# {项目名} — 进度追踪

> 最后更新：{ISO 8601 时间戳}

## 状态
<!-- STATUS: {active|paused|done} -->

## 当前任务
<!-- TASK: {任务描述} -->
<!-- TASK_STATUS: {pending|in_progress|done} -->

## 变更历史
| 时间 | 变更类型 | 描述 | Agent/人 |
|------|---------|------|----------|
| ... | ... | ... | ... |

## 待办
- [ ] ...
- [x] ...

## 阻塞项
<!-- 无则写"无" -->
```

> **关键**：`<!-- STATUS: -->` 和 `<!-- TASK: -->` 注释是 Hook 脚本的解析锚点，**不可删除或修改格式**。

## §4 GATES.md 门禁体系

```markdown
# {项目名} — 质量门禁

## 🔴 CRITICAL（不通过则不得合入）
- [ ] 代码可运行（有明确的启动命令并验证通过）
- [ ] 无语法错误
- [ ] 无硬编码密钥/令牌
- [ ] PROGRESS.md 已更新

## 🟡 IMPORTANT（不通过需注释原因）
- [ ] 新增代码有注释
- [ ] 错误处理完整
- [ ] 关联文档已更新

## 🟢 NICE（尽量满足）
- [ ] 测试覆盖新增逻辑
- [ ] 日志输出合理
```

## §5 Agent 行为准则

任何 Agent（Claude Code、Codex、Gemini 等）进入 你的工作区时：

1. **进入项目目录 → 必须先读该项目的 `CLAUDE.md`**
2. **做出任何文件变更 → 必须更新该项目的 `PROGRESS.md`**
3. **声称任务完成 → 必须先通过 `GATES.md` 🔴 全部项目**
4. **遇到缺失的三件套 → 必须先补齐再工作**
5. **不确定项目归属 → 查阅根目录 `PROJECTS.md`**

---

## §6 强制检查点绑定

| Hook 事件 | 行为 |
|-----------|------|
| `PreToolUse:Write` | 检查目标项目是否有三件套，缺失则警告 |
| `PreToolUse:Edit` | 检查目标项目是否有三件套，缺失则警告 |
| `PostToolUse:Write/Edit` | 提示更新 PROGRESS.md |
| `Stop` | 全项目 PROGRESS.md 快照保存到 checkpoint |
